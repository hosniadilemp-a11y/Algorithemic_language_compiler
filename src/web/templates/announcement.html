Minute Pédagogique : La Latence Réseau

En informatique, la latence est le temps que met une information pour voyager d'un point A à un point B. Même si les données voyagent presque à la vitesse de la lumière (dans les fibres optiques), la distance géographique joue un rôle critique.

Jusqu'à présent, notre Serveur Web (qui fait tourner le code Python) et notre Base de Données (qui stocke vos défis et scores) étaient situés sur des continents différents. À chaque fois que vous chargiez une page, le serveur devait traverser l'océan pour demander les données à la base de données, attendre la réponse, et seulement ensuite vous envoyer la page. Cela créait des lenteurs visibles.

Avant : Forte Latence

Serveur distant de la BD = Temps de réponse très lent à cause des allers-retour sur des milliers de kilomètres.

Aujourd'hui : Colocalisation

Le Serveur Web et la Base de Données sont colocalisés dans la même région (même centre de données).

Le Résultat : Le temps de trajet des données est passé de centaines de millisecondes à moins d'une milliseconde ! Le site est maintenant instantané. C'est un concept fondamental en architecture logicielle : rapprocher le calcul de la donnée.
